FAQs

These questions come from workshops, presentations, our feedback issue tracker, our mailing lists, and some from a Reddit AMA that we did a while back.

General questions

How do I cite Open Tree of Life?

To cite the synthetic tree as a whole, cite the PNAS paper. You might also want to cite the website and access date if you are referring to a particular version of the tree. If you are using a subtree from OpenTree, you should also cite the phylogenies used to create that subtree (instructions on how to do that).

Synthetic tree of life

Are there branch lengths and/or dates on the synthetic tree?
The synthetic tree does not currently summarize branch lengths or dates. The tree is constructed by an automated procedure from information found in published phylogenetic trees. These trees, as uploaded, often lack branch lengths or dates. When branch lengths are present, their meaning is not consistent across studies and it is not clear how one would calibrate them when combining trees.
How frequently do you update the synthetic tree?

The update frequency has been pretty slow up to spring 2016. Starting in March 2016, we will be able to update the tree much more frequently (at least once per month).

How do I see to a tree depth greater than 3 (e.g. for a screenshot)?
Our tree viewer it is limited to depth of three for performance reasons. As the tree becomes better resolved through addition of more phylogenies, there will be fewer polytomies (clades with thousands of taxa), so we should be able to adjust this.
Can I switch to a different tree viewer?
We only have the one tree viewer right now. The folks behind OneZoom are interested in visualizing our data, and we hope providing the data will perhaps inspire others to build much cooler visualizations than what we have currently on our site. If you want to look at the tree locally, you can download and view it in Dendroscope (which is the only one we know that can handle a tree of this size).
Why don't you have the tree from Publication X in the synthetic tree?

The tree may not be in our database of trees (either because no one has uploaded the tree, or because the authors did not publish a tree file with their publication). If the tree is in our database, then it might not be sufficently curated, or it may not be in one of the tree collections that we use for synthesis. See Questions about tree curation for related questions.

Something is wrong in the synthetic tree. How do I fix / report this?

Many things are wrong in the synthetic tree. If you don't see the relationships that you expect, the best way to correct this is to upload a tree that contains the expected relationships (or to contact the authors of such a tree and have them upload). You can also leave feedback, which helps the community of curators to know where new trees would be most useful. Note that we do not make manual changes to the tree, so leaving feedback will not cause a change without the supporting tree being in our database.

Open Tree Taxonomy

How do I add new taxon to the taxonomy?

We do not currently have an interface for adding new taxa, but we do plans to allow users to propose new taxa based on input phylogenies. The best way to see new taxa in Open Tree is to propose changes to one of our input taxonomies (NCBI, GBIF, Index Fungorum, WoRMS, etc). If you have a suggestion for what sort of evidence we could request to support the addition of new taxa, please leave us some feedback.

Adding phylogenies

How do I add new tree to the database?

Select 'Add/Browse' trees in the menu above, then Add a New Study. Anyone can upload trees from a published study. See link for documentation on the tree curation interface.

Can I add an unpublished tree?

We only take published trees into the database. This is so we have access to information about how each tree was constructed. Note that preprints and theses are ok - we just need a link to the DOI or a URL for the publication. Also note that our database of trees is publicly accessible on GitHub and we do not support infrastructure for private data.
